## 快速安装

### 方式一：纯 SKILL.md（推荐，零依赖）

```bash
# 安装到 Claude Code
mkdir -p ~/.claude/skills/leijun
cp leijun.skill/SKILL.md ~/.claude/skills/leijun/
```

重启 Claude Code 后，输入 `/leijun` 或说"用雷军的视角回答"即可触发。

### 方式二：SKILL.md + 向量检索（深度使用）

```bash
# 安装 SKILL.md
mkdir -p ~/.claude/skills/leijun-rag
cp leijun.skill-rag/SKILL.md ~/.claude/skills/leijun-rag/

# 安装依赖
cd leijun.skill-rag
python -m venv .venv
source .venv/bin/activate
pip install -r scripts/requirements.txt

# 数据入库
python scripts/ingest.py
```

然后在 Claude Desktop 配置中添加 MCP Server（见 `leijun.skill-rag/README.md`）。

## 效果示例

### 问：如何看待现在的一人公司？仅凭一人+AI，真能成功吗？

> 说实话，这个问题我一直在想。
>
> 先说结论：一人公司一定能成功，但前提是你要搞清楚"成功"的定义是什么。
>
> AI确实是一次巨大的时代红利。我说过，创业成功85%靠运气——这个运气不是天上掉馅饼，而是你有没有站在趋势上。AI就是这样一个杠杆。以前一个人做不了的事情，现在可能真的能做了。效率提升了10倍，这就是杠杆。
>
> 但我要泼一盆冷水——先想清楚怎么输。我在创办小米的时候就提醒自己：我一定会死的。一人公司最大的风险是你一个人扛不住，规模天花板，退出机制。不要被"一人公司"这个概念框住。一人公司不是目的，而是一种效率最高的起步方式。
>
> 最重要的一点：趋势不是给你躺赢的，是给你杠杆的。杠杆要用对地方，用对了事半功倍，用错了加速死亡。

运用了**顺势而为**、**效率为王**、**先想怎么输（底线思维）**、**七字诀**

### 问：如何评价鸿蒙智行的五界三境，这种模式为何能成功，未来前景怎么样？

> 说实话，这个问题挺敏感的。但我是做产品的人，讲的是客观事实。
>
> 华为的"五界三境"本质是什么？是华为的技术输出+品牌赋能+渠道加持，跟车企合作造车。小米呢？小米是自己干。3400名工程师，超100亿研发，从零开始造车。这两种模式，各有各的道理。
>
> 华为模式能成功有三个原因：第一，顺势而为——抓住了智能汽车这个大趋势，手里有鸿蒙操作系统和智能驾驶技术两张王牌。第二，效率高——轻资产模式，不用自己建工厂。第三，品牌势能——消费者心中"技术"两个字可以直接迁移到车上。
>
> 但这种模式也有风险：品控和口碑的连带效应、合作伙伴的利益博弈、能不能真正形成闭环。最终比的是什么？不是谁的模式更花哨，而是谁能真正做出超出用户预期的产品。

运用了**顺势而为**、**效率为王**、**超预期口碑法则**、**对角线战略**

### 问：小米一口气布局SoC、汽车、具身智能、大模型，不怕步子迈太大了吗？

> 很多人都觉得小米步子迈太大了。但我要先讲一个五年前的故事。
>
> 2020年小米十周年，年收入突破2000亿，跻身世界500强。外人看来很成功，但我内心充满焦虑——小米虽然大了，但还不够强。我们做了十几年手机，很多核心技术还是依赖供应商。我们做了一次触及灵魂深处的大反思，结论很简单：小米必须掌握核心技术，否则永远是给别人打工的。
>
> 我们不是"什么都做"，而是围绕一个核心——人车家全生态——在做技术纵深。芯片是操作系统的基础，汽车是生态闭环的关键，大模型是AI能力的底座，具身智能是未来的延伸。看起来分散，但底层是相通的。
>
> 但我也不想粉饰太平。同时做这么多事情，每一件都是高难度，失败的概率不低。我是一个挺怕输的人，所以我做所有事情之前，先想我会怎么输。这叫底线思维。伟大是熬出来的，小米才15岁，还年轻。急什么？

运用了**顺势而为**、**十倍投入原则**、**底线思维**、**长期主义**、**技术为本**

更多对话示例在 [`leijun.skill/examples/demo-conversation.md`](leijun.skill/examples/demo-conversation.md)。

## 两个版本

| | leijun.skill | leijun.skill-rag |
|---|---|---|
| **SKILL.md** | 纯人设 | 人设 + 检索指令 |
| **知识来源** | 精炼在 SKILL.md | SKILL.md + 1038 个向量文档 |
| **依赖** | 无 | Python + ChromaDB + Embedding API |
| **触发方式** | `/leijun` | `/leijun-rag` |
| **检索** | 不支持 | 自动检索原始材料做事实支撑 |

两个版本独立完整，不互相依赖。leijun.skill 是 leijun.skill-rag 的子集。选择取决于你的使用场景：需要开箱即用选前者，需要引用原文做事实支撑选后者。

| 模型 | 一句话 | 交叉验证 |

       → 整合表达 DNA + 决策案例 + 人生时间线
